在Github上用Markdown创建技术博客

一、实验目的

1.熟悉Github网站
2.掌握Markdown语法

二、实验任务

1.注册Github账号
2.创建Gihub Pages repository
3.发布第一篇博客
4.学习Markdown
5.利用Hexo生成博客并部署到Github

三、实验步骤与结果

1..安装git
2.安装node
3.设置npm镜像为淘宝
npm config set registry https://registry.npm.taobao.org
4.安装hexo
npm install -g hexo-cli
5.初始化站点
1.hexo init myhexo 2.cd myhexo 3.npm install.
6.浏览本地站点
hexo server
7.在Github上新建repository，名称必须是用户名.github.io
8.修改配置文件_config.yml的deploy部分
9.安装 hexo-deployer-git
npm install hexo-deployer-git --save
10.配置git全局邮箱与名字
git config --global user.email "shj.zhu@gmail.com" git config --global user.name "Shaojun Zhu"
11.生成博客
hexo generate
12.部署博客
hexo deploy

1.Fork他的仓库：这是GitHub操作，这个操作会复制对方的仓库。
2.Clone你的仓库：这是Git操作。。现在这个仓库就会存储在你本地计算机上。
3.更新某些文件：提交你的更改：这是Git操作。使用该操作让你发送”记录我的更改”的命令至GitHub。在本地计算机上完成。
4.将你的更改push到你的GitHub仓库：给对方发送一个pull request：如果你认为Joe会接受你的修改，你就可以给他发送一个pull request。如果Joe接受了你的pull request，他将把那些修改拉到自己的仓库。

四、实验小结

通过对github的学习，更加了解如何高效的搭建自己的开发环境，搭建出自己的环境自己也很开心，了解了更多开发环境的知识，更熟悉了操作系统和程序环境，学习到了很多。
